## Implementation plan (do this)

### A) L1 lease foundation
1. Document L1 fully in `.agents/rules/isolation.md`:
   - Lease file path: `.agents/locks/{job_id}.lease.json`
   - Schema fields: `job_id`, `owned_paths` (string array), `status` (`running`|`released`), `acquired_at` (ISO-8601), optional `type`
   - Rule: if any lease with `status=running` has overlapping path prefix with requested owned_paths → refuse spawn
   - Path overlap = exact match OR prefix match (normalize to repo-relative `/` or `\`)
   - L0 `write-job.lock` remains for single-writer default; L1 is additional ownership when parallel writers are considered later. For now Operator still uses L0; L1 is foundation.

2. Create `scripts/lease-paths.ps1`:
   - Params: `-Action acquire|check|release`, `-JobId`, `-OwnedPaths string[]` (required for acquire/check)
   - acquire: refuse if overlap with any running lease; write `{job_id}.lease.json`; exit non-zero on refuse
   - check: print overlapping job ids if any; exit 1 if overlap, 0 if free
   - release: set status released or delete lease file for JobId
   - PowerShell 5.1 compatible (`#requires -Version 5.1`)
   - No external modules

3. Optional minimal hook in `scripts/delegate-codex.ps1`:
   - Add optional `[string[]]$OwnedPaths = @()`
   - If write job AND OwnedPaths non-empty: call lease acquire before exec; release in finally (in addition to existing write-job.lock)
   - If OwnedPaths empty: behavior unchanged (L0 only)
   - Do not break existing smoke-001 usage

4. Update `.agents/skills/codex-system/SKILL.md` with short L1 note: declare owned_paths for implement/fix; run lease-paths before spawn when using L1.
