### Grok OS sandbox docs (local `~/.grok/docs/user-guide/18-sandbox.md`)
- Profiles: `off`, `workspace`, `devbox`, `read-only`, `strict`
- Mechanism: Landlock (Linux) / Seatbelt (macOS) — **Windows not listed**
- Fail-open if sandbox cannot apply (except custom profile with deny)

### Current Codex adapter surface (`scripts/delegate-codex.ps1`)
Ops already present:
1. **resolve invocation** — `Resolve-CodexNodeInvocation` (prefer node+codex.js else `codex`)
2. **map sandbox** — write types → `workspace-write`; else `read-only`
3. **pass prompt** — pipe Prompt Contract text on stdin to `codex exec`
4. **capture result** — `.agents/logs/codex/<id>.{last,stderr,combined}.txt` via `-o` + stream capture
5. **classify exit** — nonzero `$LASTEXITCODE` → Write-Error; empty last message → error

Plus harness concerns outside pure runtime adapter: L0 write lock, L1 leases, L2 worktree, Prompt Contract headings check.
